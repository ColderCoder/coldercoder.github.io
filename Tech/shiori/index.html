Shiori 网络书签

ColderCoder
2020-12-23
Tech
heroku

懒癌作祟，很想要一个read-later解决方案，但又对pocket之流提不起兴致，要是源站挂了，免费版用户保存的书签也随之404，而付费订阅的价格对轻度用户来说又有点离谱。类似于WayBack Machine的服务也不适合个人收藏阅读。

Shiori是个不错的解决方案，用户能够自行部署类pocket的应用。

项目仓库的README还提供了Heroku一键部署按钮，十分方便。
（heroku登录地址可能在中国大陆被封锁）

Heroku会定期重启Dyno，届时所有数据将被重置，使用PostgreSQL以解决问题：

在 Resource-Add-ons 中添加 Heroku Postgres，在 Postgres 的 Settings 下找到 Database Credentials，并参照这个指南在项目的 Setting - Config Vars 下填写相应字段。

Web界面的初始账号密码是 shiori/gopher，登陆后创建用户，新建的第一个用户为 owner，同时初始账号被禁用。

Shiori提供了Chrome/Firefox插件，Firefox在release页面点击.xpi文件即可安装；Chrome下载Source Code后解压，扩展程序-开发者模式-加载已解压的扩展程序。

安装插件后在扩展程序选项中填写服务端地址、用户名、密码，完成配置。
